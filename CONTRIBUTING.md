# Contributing to HORUS Tools

First off, thank you for considering contributing to HORUS Tools! 🚒

## How Can I Contribute?

### Reporting Bugs

- Use the [Bug Report](https://github.com/isaratech/horus-tools/issues/new?template=bug_report.md) issue template.
- Include steps to reproduce the issue.
- Mention your browser and OS.

### Suggesting Features

- Use the [Feature Request](https://github.com/isaratech/horus-tools/issues/new?template=feature_request.md) issue template.
- Describe the use case and expected behavior.

### Submitting a Pull Request

1. **Fork** the repository and create a branch from `main`.
2. **Install** dependencies: `npm install`
3. **Develop**: `npm start` (runs on `http://localhost:4200/`)
4. **Test** your changes: `npm test`
5. Make sure the build passes: `npm run build`
6. Open a **Pull Request** against `main` with a clear description.

### Code Style

- Follow the existing code conventions (Angular standalone components, Tailwind CSS utilities).
- Use **TypeScript** strict mode.
- Keep components small and focused on a single tool.
- Write meaningful commit messages.

### Commit Messages

Use clear, descriptive commit messages:

```
feat: add new hydraulic calculator tool
fix: correct SCBA autonomy calculation
docs: update README with new tool description
```

## Code of Conduct

This project follows a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold it.

## Questions?

Feel free to open a [Discussion](https://github.com/isaratech/horus-tools/discussions) or an issue if you have any questions.
